# Feature7 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a new `payment-service` Maven module integrated with the reviewed `product-service` over `RestTemplate`, with product lookup, payment execution, real balance debit, payment persistence, and stable client-facing errors.

**Architecture:** The repository becomes a parent Maven project with `product-service` and `payment-service` modules. Existing source code and tests move into `product-service` for minimal churn and backward compatibility, while `payment-service` is added as a new Spring Boot app that uses a dedicated HTTP client bean to call `product-service`. Successful debit happens in `product-service`, then successful payment persistence happens in `payment-service`.

**Tech Stack:** Java 17, Spring Boot 3.3.5, Spring Web, Spring Data JPA, PostgreSQL, Flyway, RestTemplate, JUnit 5, Mockito, MockMvc.

public Product debit(Long productId, BigDecimal amount) {

public record PaymentResponse(
        Long paymentId,
        Long userId,
        Long productId,
        BigDecimal amount,
        String description,
        LocalDateTime createdAt

}

@Component
public class PaymentMapper {
    public Payment toEntity(Long userId, Long productId, BigDecimal amount, String description, LocalDateTime createdAt) {
        return new Payment(null, userId, productId, amount, description, createdAt);

    public PaymentResponse toResponse(Payment payment) {
        return new PaymentResponse(
                payment.getId(),
                payment.getUserId(),
                payment.getProductId(),
                payment.getAmount(),
                payment.getDescription(),
                payment.getCreatedAt()
        );
    }
}

public class ProductClient {

    public List<ProductClientProductResponse> getProductsByUserId(Long userId) {

@Transactional
public PaymentResponse execute(PaymentRequest request) {
    validateRequest(request);
    ProductClientProductResponse product = productClient.getProductById(request.productId());
    if (!product.userId().equals(request.userId())) {
        throw new ProductOwnershipException("Product with id=" + request.productId() + " does not belong to user with id=" + request.userId());

    productClient.debitProduct(request.productId(), request.amount());

                    request.amount(),
                    request.description(),
                    LocalDateTime.now()
            )
    );
    return paymentMapper.toResponse(payment);
